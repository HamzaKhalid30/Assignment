# A6: Discover Cryptographic Implementation Used Offline

## Overview

While exploring offline systems, I identified that many devices use disk encryption to protect stored data. Disk encryption is a cryptographic technique that secures data at rest by converting it into unreadable form using encryption algorithms such as AES (Advanced Encryption Standard). This ensures that even if a device is lost or stolen, the data cannot be accessed without proper authentication.

## Practical

2. Right-click → Run as Administrator (if possible)

## Findings and Details

- I discovered that offline systems use Full Disk Encryption (FDE) to protect stored data such as files, documents, and system information.
- Disk encryption works by encrypting every bit of data on the storage device, ensuring unauthorized users cannot read it without the correct key.
- Most systems use AES encryption, which is a strong symmetric encryption algorithm widely used for securing data.
- Encryption happens automatically in the background, meaning data is encrypted when written to disk and decrypted when accessed by authorized users.
- This protects data at rest, which is especially important if a laptop, USB drive, or hard disk is lost or stolen.
- Examples of offline cryptographic tools include BitLocker (Windows), FileVault (Mac), and LUKS (Linux).

## Reflection

This activity helped me understand how cryptography is used not only online but also offline to protect stored data. Disk encryption ensures that even if someone gains physical access to a device, they cannot access sensitive information. It highlights the importance of securing data at rest using strong encryption techniques like AES.
