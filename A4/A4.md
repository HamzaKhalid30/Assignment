# A4: Discover a Vulnerable Website (HTML Injection)

## Overview

While performing web application testing, I identified a vulnerability known as **HTML Injection**. This type of vulnerability occurs when a website fails to properly validate user input, allowing attackers to inject HTML code into web pages. This can modify how the page is displayed and potentially mislead users.


## Practical

1. Open Google and use a dork query:
   ```
   inurl:index.php?msg=
   ```

2. From the results, select a website that accepts user input via URL parameters.

3. Modify the URL by adding a custom payload:
   ```
   http://[REDACTED]/admin/index.php?msg=hacker"><h1>test</h1>
   ```

4. Load the URL in the browser.

5. Observe the output if the injected HTML is rendered (e.g., *"test"* appears as a heading), the site is vulnerable.

## Findings and Details

- I discovered that the website was directly reflecting user input from the URL parameter (`msg=`) without proper validation.
- By injecting the payload `<h1>test</h1>`, the application rendered it as actual HTML, confirming the vulnerability.
- This type of vulnerability is known as **Reflected HTML Injection**, where malicious input is immediately displayed in the response.
- The root cause is **lack of input sanitization and output encoding**, allowing attackers to manipulate webpage content.
- Attackers can use this vulnerability to perform actions such as:
  - Webpage defacement
  - Phishing attacks
  - Tricking users into trusting malicious content
- HTML Injection is similar to XSS but typically involves injecting HTML tags instead of scripts, still posing significant security risks.



## Reflection

This activity helped me understand how simple input fields in URLs can become serious vulnerabilities if not properly secured. Even a basic payload like `<h1>` can confirm a weakness in the system. It also showed how attackers use techniques like **Google Dorking** to find vulnerable targets easily. Proper input validation and secure coding practices are essential to prevent such attacks.
