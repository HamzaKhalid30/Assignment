# A1: Discover Security Concepts Used on Campus

## Overview

During my observation of the campus network infrastructure, I focused specifically on wireless security implementation rather than physical controls. I identified that the campus uses a segmented Wi-Fi architecture, which plays a crucial role in maintaining security and access control across different user groups.

## Findings and Details

- The campus Wi-Fi is divided into multiple access levels, primarily a public network for students and a restricted/private network for faculty and administrative staff.
- The student Wi-Fi (low-privilege network) provides only internet access and does not allow communication with internal systems such as administrative servers or databases.
- The staff Wi-Fi (high-privilege network) is isolated and provides access to sensitive systems like internal portals, file servers, and academic records.
- This separation follows the concept of network segmentation and access control, ensuring that even if a student device is compromised, it cannot directly affect critical infrastructure.
- Such segmentation reduces the attack surface and prevents lateral movement inside the network, which is a common technique used by attackers.
- This model aligns with modern Wi-Fi security practices where authentication, encryption, and access control are combined to protect networks.

## Reflection

This activity helped me understand how organizations implement privilege-based network isolation to enhance security. Instead of relying on a single network, dividing access based on user roles significantly improves protection against internal and external threats.
