# A5: Discover Cryptographic Implementation Used Online

## Overview

While exploring online platforms, I identified that most modern websites use HTTPS (HyperText Transfer Protocol Secure), which is powered by a cryptographic protocol called TLS (Transport Layer Security). This ensures that communication between the user and the website is encrypted and secure from attackers.

## Practical

1. Open any website such as `https://google.com`
2. Check the padlock icon in the browser address bar.
3. Click on the padlock → View Connection is secure / Certificate details.
4. Inspect details such as:
   - TLS version (e.g., TLS 1.2 / TLS 1.3)
   - Certificate issuer
   - Encryption algorithm
5. Open Developer Tools → Network tab → Reload page → Observe HTTPS requests.

## Findings and Details

- I discovered that websites use TLS encryption to secure communication between client (browser) and server.
- TLS ensures that sensitive data such as passwords, credit card details, and personal information cannot be intercepted by attackers.
- The protocol uses a combination of asymmetric encryption (for key exchange) and symmetric encryption (for data transfer), providing both security and performance.
- During the connection process, a TLS handshake occurs where both client and server establish a secure session and exchange cryptographic keys.
- The presence of HTTPS ensures:
  - **Confidentiality** (data is encrypted)
  - **Integrity** (data cannot be modified)
  - **Authentication** (verifies website identity)
- This cryptographic implementation is widely used across websites, APIs, email services, and other online communications.

## Reflection

This activity helped me understand how cryptography is applied in real-world internet usage. The HTTPS protocol ensures secure communication by encrypting data and verifying identities. It showed me that even everyday browsing relies heavily on strong cryptographic mechanisms like TLS to protect users from attacks such as eavesdropping and man-in-the-middle attacks.
